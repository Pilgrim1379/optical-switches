Optical Switches

A guide to tests, calibrations & the beam-steering control loop

How an optical switch works, how it is calibrated and tested, and how its control loop continuously maximises optical performance.

1

What is an optical switch?

An optical switch dynamically connects any input fibre to any output fibre under electronic control. Light is routed across free space by steerable elements — there is no continuous fibre between input and output.

Two free-space architectures are used: 3D-MEMS deflects the beam with tiny movable micro-mirrors, while piezoelectric beam-steering points collimated beams using piezoelectric actuators.

Which technology?

Both free-space architectures — 3D-MEMS micro-mirrors and piezoelectric beam-steering — share the same performance metrics and tests. This guide explains the steering-mirror control loop, which applies to both.

Typical uses: data centres, test systems, ROADM, network protection, instrumentation.

2

The optical path

Light travels from the input fibre to the output fibre via collimating lenses and two steerable mirrors.

  1. Laser / source — light enters the switch.
  2. Input fibre — carries light to the switch.
  3. Collimating lens (input) — expands light into a parallel beam.
  4. Input steering mirror — steers the beam in two axes (X, Y).
  5. Free-space beam path — light travels through the air.
  6. Output steering mirror — steers the beam toward the output.
  7. Collimating lens (output) — focuses the beam into the fibre core.
  8. Output fibre — carries light out of the switch.
  9. Photodetector (internal) — measures optical power for feedback.
3

Key performance metrics

How to read them

IL — Insertion Loss

Power lost in the forward direction.

IL(dB) = 10·log₁₀(Pin / Pout)
Lower is better

Lower IL means less power is lost through the switch.

ORL — Optical Return Loss

Reflected power coming back toward the source.

ORL(dB) = −10·log₁₀(Pref / Pin)
Higher is better

Higher ORL means less reflected light — almost none comes back.

XTALK — Crosstalk

Unwanted light appearing at non-selected outputs.

XTALK(dB) = −10·log₁₀(Pleak / Psignal)
Higher is better

Higher XTALK means less leakage to other outputs.

Typical good values

Rule of thumb — depends on product.

IL

  • ≤ 1.0 dB typical
  • ≤ 1.5 dB maximum
  • Some applications may allow more.

ORL

  • ≥ 50 dB typical
  • ≥ 60 dB high-performance

XTALK

  • ≥ 60 dB typical
  • ≥ 70 dB advanced systems

Always refer to the specific datasheet for guaranteed specifications.

4

The beam-steering control loop

The heart of the switch

A closed-loop servo that holds each beam-steering element on the target position for the requested connection. Piezoelectric actuators tip and tilt the input and output elements so their collimated beams meet across the free-space gap — and the loop keeps them there.

  1. DSP looks up the target actuator positions for the requested input→output connection from the calibration table.
  2. DACs convert those targets into drive voltages for the piezoelectric actuators.
  3. The actuators tip and tilt the input and output beam-steering elements so the two collimated beams meet across the gap.
  4. Integrated position sensors report each actuator's true position back to the DSP.
  5. The DSP servos each actuator onto its target, continuously correcting: ↑ on target → hold ↓ drifted off → nudge back
  6. Alignment spans 4 axes (Input X, Input Y, Output X, Output Y) — each held by the servo.
  7. Because the loop holds position, not power, the path stays aligned even with no light present: dark fibres can be pre-provisioned, and switching is independent of wavelength, power and direction. An Optical Power Monitor (OPM) can peak up and verify the path.

How the target is found — then held

During calibration the beam is swept to find the peak (maximum power) and its position is recorded. In operation the position servo parks the beam on that recorded spot and holds it there.

Holds alignment over temperature, drift & vibration

Keeps alignment even on dark (unlit) fibre

Switching independent of wavelength, power & direction

Delivers repeatable, low-loss connections

Enables advanced diagnostics & testing

5

Tests & calibrations explained

What each one does, what it checks, and what it relates to

1

Connectivity Check

Verifies every requested input–output path can be established; checks mirror movement, controller operation, addressing and routing.

Detects: stuck mirrors, dead slices, actuator failures, addressing faults.

Relates to: Power Grid, IL, ORL, Stability (if this fails, others are unreliable).

2

Power Grid Check (Power Calibration)

Measures optical power across a range of mirror positions to find the optimum (maximum) power point for every path.

Detects: misalignment, weak mirrors, dirty optics, detector issues, damage.

Relates to: Thermal Cal, IL, ORL, Crosstalk, Stability.

3

Thermal Calibration

Characterises how mirror positions change with temperature and builds compensation tables.

Purpose: keeps mirrors at the optimum as temperature changes.

Relates to: Power Grid, Stability, IL over temperature.

4

Open Cal Log

Opens the calibration history log — past calibrations, results, warnings, failures, dates.

Purpose: troubleshooting and traceability.

Relates to: all tests (helps diagnose trends).

5

Power Measurement

Live optical-power monitoring from internal detectors, used during debugging, calibration and alignment.

Note: not a qualification test.

Relates to: Power Grid, Stability, IL.

6

Splitter / Detector Calibration

Calibrates internal photodetectors so their readings match actual optical power.

Purpose: ensures Power Grid and other measurements are accurate.

Relates to: Power Grid, Power Measurement, IL.

7

DAC Range Grid Test

Verifies DAC output range and linearity so mirrors can be driven over the full range without clipping or saturation.

Detects: bad DACs, incorrect bias, driver or actuator issues.

Relates to: Power Grid, Connectivity, Stability.

8

Dynamic X-Coupling

Measures how moving one mirror affects neighbouring mirrors — mechanical coupling that causes alignment errors, instability and higher IL.

Detects: cross-axis mechanical coupling.

Relates to: Stability, IL, Power Grid.

9

Stability Test

Measures the same path repeatedly over time to ensure power remains stable.

Detects: thermal drift, actuator drift, vibration, unstable control loops.

Relates to: Thermal Cal, Power Grid, Reliability.

10

Switching Speed

Measures how fast the switch changes from one connection to another and settles to a stable, in-spec power level.

Important for: high-speed and protection-switching applications.

Relates to: Control Loop, Driver Electronics.

11

XTALK with Parks

Measures optical crosstalk (leakage) at non-selected outputs while the desired path is active.

Detects: leakage, misalignment, reflections, poor isolation.

Relates to: IL, ORL, Power Grid.

12

Noise vs Detuning Test

Detunes a mirror away from the optimum position and measures power and noise.

Purpose: evaluates sensitivity and control robustness.

Relates to: Power Grid, Control-Loop Tuning, Stability.

13

Power Cycling Test

Repeatedly powers the switch off and on and verifies it still meets specification; checks calibration retention and startup reliability.

Detects: retention and startup faults.

Relates to: EEPROM, Startup, Thermal, Overall Reliability.

14

IL & ORL Test

Full optical performance test — measures insertion loss and optical return loss for all paths. The final qualification test.

Purpose: final sign-off of optical performance.

Relates to: Power Grid, Thermal, Alignment.

Typical test order

  1. Connectivity Check
  2. DAC Range Test
  3. Splitter / Detector Calibration
  4. Thermal Calibration
  5. Power Grid Check
  6. IL & ORL Test
  7. Stability, XTALK, Speed, etc.
  8. Power Cycling Test

Order may vary by product or process.

7

Understanding results

Good switch

  • Connectivity pass
  • Power Grid pass
  • IL low
  • ORL high
  • XTALK low (high dB)
  • Stable over time
  • Fast switching

→ High performance, reliable operation

Connectivity pass, IL fail

Mirrors move correctly, but alignment is not optimal. Check Power Grid, thermal drift and optical cleanliness.

Possible causes

  • Misalignment
  • Dirty optics
  • Wrong calibration

Power Grid fail

Cannot find a strong power peak.

Possible causes

  • Dirty optics
  • Poor detectors
  • Damaged mirrors
  • Mechanical issues

Good Power Grid, stability fail

Initial alignment good, but drifts over time.

Possible causes

  • Thermal-calibration drift
  • Environment
  • Power supplies

Remember

  • Calibration teaches the switch where the optimum is.
  • The control loop keeps it there.
  • Tests verify performance, reliability and robustness.

Units & conversions

  • IL, ORL and XTALK are in dB (logarithmic).
  • Power: 1 mW = 0 dBm
  • 0 dB = no loss, reflection or crosstalk.
  • Every 3 dB = half (or double) the power.

Helpful analogy

Think of a hill where height is optical power. Calibration is the one-time climb that finds the summit and marks it on the map (the lookup table). In operation the switch doesn't re-climb — position sensors keep the beam parked on that exact summit, holding it there even as conditions (temperature, drift, vibration) change. That is what the control loop does.
